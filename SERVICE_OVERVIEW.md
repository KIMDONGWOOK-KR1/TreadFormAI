## 1. 서비스 개요

### 1.1 한 줄 정의

스마트폰 앱으로 트레드밀 측면 러닝 영상을 촬영하면, **부상 방지 3대 지표**를 분석하여 **한국어 코칭 피드백**을 제공하는 모바일 자세 분석 서비스.

### 1.2 누구를 위한 서비스인가

**피트니스 센터 트레이너**가 회원의 자세를 객관적 데이터로 코칭할 수 있게 돕고, 동시에 **일반 러너**가 본인의 부상 위험을 스스로 점검할 수 있게 하는 듀얼 모드 앱입니다.

### 1.3 어떻게 작동하는가

```
1. 트레드밀 측면에 스마트폰을 거치하고 5~10초 촬영
        ↓
2. 앱이 영상을 자동으로 720p로 변환 후 분석 서버에 업로드
        ↓
3. AI가 부상 방지 3대 지표를 분석 (약 30초 소요)
        ↓
4. 스켈레톤 오버레이 영상 + 한국어 코칭 피드백 제공
        ↓
5. 위험 구간 자동 슬로우 모션 + 비포/애프터 누적 그래프
```

### 1.4 왜 만들어졌는가

국내 피트니스 센터에서 자세 코칭은 트레이너의 **육안 판단**에 의존하고, 객관적 자세 분석은 **대학병원·스포츠 의학 연구실**에서만 가능했습니다. 글로벌 솔루션이 존재하지만 모두 영문 UI이고, 일반인이 이해하기 어려운 정보 과부하 또는 분석 지연 등의 문제가 있었습니다.

TreadForm은 **한국인을 위한, 트레이너와 일반인 모두가 직관적으로 사용할 수 있는** 자세 분석 도구로 이 갭을 메웁니다.

## 2. 해결하려는 문제

### 2.1 트레이너의 페인 포인트

> *"회원에게 자세 문제를 객관적 수치로 보여줄 도구가 없습니다."*

- 육안 판단에 의존한 코칭 → 회원 설득력 부족
- PT 효과를 데이터로 증명할 방법이 없음
- 회원 재등록 유도 시 정성적 변화만 강조 가능

### 2.2 일반 러너의 페인 포인트

> *"내 러닝 자세가 부상 위험이 있는지 알 방법이 없습니다."*

- 인터넷 영상으로 자가 진단 → 자기 자세 객관적 평가 불가
- 한 번 부상을 겪고 나서야 자세의 중요성을 인지
- 스포츠 의학 분석은 비용과 접근성이 부담

### 2.3 피트니스 센터의 페인 포인트

> *"PT의 가치를 시각적·수치적으로 증명할 차별화 도구가 부족합니다."*

- 타 센터와의 차별성 부각 어려움
- 회원 retention을 위한 데이터 기반 도구 부재
- PT 비용 정당화에 필요한 객관적 결과물 부재

## 3. 타겟 사용자

### 3.1 1차 타겟 — 피트니스 센터 트레이너 (B2B)

| 항목 | 내용 |
|---|---|
| 연령 | 25~45세 |
| 직무 | 퍼스널 트레이너 (PT) |
| 핵심 니즈 | 회원 자세 코칭의 객관성 + PT 효과 증명 |
| 사용 시나리오 | 회원과 함께 트레드밀 옆에서 촬영 → 결과 보며 코칭 |
| 가치 제안 | "트레이너의 PT 가치를 시각적으로 증명하세요" |

### 3.2 2차 타겟 — 일반 러너 (B2C)

| 항목 | 내용 |
|---|---|
| 연령 | 20~50세 |
| 사용처 | 헬스장 트레드밀, 가정용 트레드밀 |
| 핵심 니즈 | 자가 자세 점검, 부상 예방 |
| 사용 시나리오 | 운동 중 1회 촬영 → 결과 보고 자세 교정 |
| 가치 제안 | "내 자세가 안전한지 직접 확인하세요" |

### 3.3 3차 타겟 — 교내 체육 시설 / 육상 연구실

| 항목 | 내용 |
|---|---|
| 사용처 | 대학 체육관, 스포츠 학과 연구실 |
| 핵심 니즈 | 학생 선수 자세 데이터 누적, 연구용 |
| 사용 시나리오 | 학생별 정기 측정 → CSV 데이터로 분석 |
| 가치 제안 | "학생 선수의 자세를 데이터로 관리하세요" |

## 4. 핵심 가치 제안

### 4.1 운동 시간 내 즉시 피드백

분석에 5영업일이 걸리는 경쟁사와 달리, **약 30초 만에 분석 완료**. 운동 시간(1~1.5시간) 내에 결과를 보며 즉시 자세를 교정할 수 있습니다.

### 4.2 100% 한국어

UI, 리포트, AI 코칭 메시지 모두 자연스러운 한국어로 제공합니다.

> *"뒤꿈치 착지가 15회로 다소 많이 발생했습니다. 무릎 부담을 줄이려면 보폭을 조금 줄여보세요."*

### 4.3 한 앱, 두 가지 모드

- **일반 모드**: 개인 러너용
- **트레이너 모드**: 회원 등록/관리 + 누적 PT 효과 그래프

별도 계정이나 별도 앱 없이 토글 하나로 전환됩니다.

### 4.4 직관적 시각화 (정보가 아닌 신호)

- 경쟁사: 15개 이상 메트릭 + 수치 점수 → 정보 과부하
- TreadForm: **부상 방지 3대 지표** + **🔴🟡🟢 색상 코딩**
- 위험 구간은 **빨간 마커 + 자동 0.5배속 슬로우 모션**으로 즉시 강조

## 5. 주요 기능

### 5.1 AR 가이드라인 카메라

스마트폰 카메라 화면에 **반투명 가이드라인**을 표시하여 누가 찍어도 동일한 측면 구도와 거리로 촬영할 수 있게 합니다.

**구성 요소**
- 트레드밀 벨트 선 가이드 (가로 점선)
- 골반 위치 박스 가이드 (중앙 사각형)
- 한국어 안내 텍스트: *"트레드밀 벨트에 맞춰주세요"*

**효과**: 표준화된 촬영 구도 → 분석 정확도 향상

### 5.2 자동 영상 최적화

촬영된 영상을 자동으로 **720p로 다운스케일** + **압축** 처리하여 업로드 속도를 5배 이상 개선합니다.

| 구분 | 원본 | 최적화 후 |
|---|---|---|
| 해상도 | 1080p | 720p |
| 파일 크기 | 150~300MB | 50~100MB |
| 업로드 시간 (WiFi) | 12~24초 | 4~8초 |

### 5.3 AI 자세 분석 (서버)

**MediaPipe Pose** AI 모델로 33개 신체 관절을 추출하고, 후처리 분석으로 부상 방지 3대 지표를 산출합니다.

**처리 단계**
1. 33개 관절 좌표 시계열 추출
2. 가림(Occlusion) 보정 (Forward Fill)
3. 좌표 흔들림 보정 (EMA 필터)
4. 좌/우 발목 독립 추적으로 착지 시점 판별
5. 3대 지표 + 좌우 비대칭 계산
6. 한국어 코칭 메시지 생성

### 5.4 결과 영상 시각화

분석 완료 후 원본 영상 위에 **3-Layer 오버레이**가 적용된 결과 영상을 제공합니다.

| Layer | 내용 |
|---|---|
| Layer 1 | 원본 러닝 영상 |
| Layer 2 | 33개 관절 스켈레톤 라인 (상태별 색상 동적 변경) |
| Layer 3 | 현재 무릎 각도 수치 + 한국어 상태 텍스트 |

### 5.5 Danger 자동 북마크 + 슬로우 모션

위험 자세가 감지된 시점이 자동으로 **빨간 마커**로 타임라인에 표시되고, 해당 구간 진입 시 **자동 0.5배속**으로 재생됩니다.

**동작 방식**
- Danger 시점 ±0.5초 구간 자동 슬로우 모션
- "🐢 0.5x 슬로우 모션" 뱃지 표시
- 트레이너가 회원에게 즉시 문제 순간을 보여줄 수 있음

### 5.6 한국어 AI 코칭 피드백

분석 결과를 토대로 가장 심각한 문제를 우선순위화하여 **자연스러운 한국어 코칭 메시지**를 생성합니다.

**예시 메시지**
- *"뒤꿈치 착지가 15회로 다소 많이 발생했습니다. 무릎 부담을 줄이려면 보폭을 조금 줄여보세요."*
- *"좌우 비대칭(12%)이 감지됩니다. 편측 부상 주의가 필요합니다."*
- *"중족 착지가 안정적으로 이루어지고 있습니다 👍"*

### 5.7 일반 사용자 모드

별도 로그인이나 회원 등록 없이 **앱 설치 직후 바로** 사용할 수 있는 개인 자가 진단 모드입니다. 트레이너 모드 토글이 OFF 상태(기본값)일 때 활성화됩니다.

**일반 사용자 모드 기능**
- 회원 등록·선택 단계 없이 즉시 촬영·분석 가능
- AR 가이드라인 카메라로 직접 촬영하거나, **갤러리·파일 앱에서 기존 영상 불러오기** 지원
- 단발성 자가 진단 워크플로우 (촬영 → 분석 → 결과 확인 1회 완결)
- 회원 ID 없이 분석 (회원 데이터에 묶이지 않음 → 익명 분석)
- 결과 영상·코칭 메시지·CSV 리포트는 트레이너 모드와 **동일한 품질**로 제공

**일반 모드 vs 트레이너 모드 비교**

| 항목 | 일반 사용자 모드 | 트레이너 모드 |
|---|---|---|
| 진입 장벽 | 설치 후 즉시 사용 | 트레이너 모드 토글 ON |
| 회원 관리 | ❌ (불필요) | ✅ 회원 등록·선택 |
| 영상 소스 | 촬영 / 갤러리 / 파일 | 촬영 / 갤러리 / 파일 |
| 분석 결과 | ✅ 동일 품질 | ✅ 동일 품질 |
| 누적 대시보드 | ❌ | ✅ 회원별 시계열 그래프 |
| 결과 공유 | ✅ 일반 공유 링크 | ✅ 비밀번호 옵션 공유 링크 |
| 주 용도 | 자가 자세 점검 | 회원 코칭·PT 효과 증명 |

**전환 방식**: 홈 화면 상단의 **"트레이너 모드"** 스위치 하나로 양방향 전환됩니다. 별도 로그아웃·재설치 없이 동일 앱에서 두 역할을 오갈 수 있어, 트레이너 본인의 러닝도 같은 앱으로 측정 가능합니다.

### 5.8 트레이너 모드

앱 내 토글로 트레이너 전용 기능을 활성화합니다.

**트레이너 모드 기능**
- 회원 등록 및 관리
- 회원별 영상 업로드 (자동 분류)
- 회원별 분석 이력 관리
- 비포/애프터 누적 그래프
- PT 효과 요약 텍스트 자동 생성

### 5.9 비포/애프터 누적 대시보드

회원별 시간에 따른 자세 변화를 그래프로 시각화합니다.

**제공 그래프 (3종)**
- 뒤꿈치 착지 발생 횟수 추이
- 평균 무릎 굴곡 각도 변화
- 수직 진폭 변화

**PT 효과 요약 예시**
> *"첫 회 15회 → 최근 3회로 뒤꿈치 착지가 감소했습니다."*

### 5.10 결과 공유

분석 결과를 카카오톡, 이메일 등으로 공유할 수 있습니다. 트레이너 모드에서는 **비밀번호 옵션이 있는 공유 링크**도 생성 가능합니다.

### 5.11 CSV 리포트 다운로드

프레임/시간대별 모든 지표가 기록된 CSV 파일을 다운로드할 수 있습니다. 트레이너의 객관적 코칭 자료 또는 연구실의 학술 데이터로 활용 가능합니다.

## 6. 핵심 분석 지표

### 6.1 부상 방지 3대 지표

#### 지표 1: 무릎 굴곡 각도 (Knee Flexion)

**측정 방식**: 골반-무릎-발목 세 관절이 이루는 각도

**왜 중요한가**: 착지 시 무릎이 적절히 굽혀져야 충격을 흡수할 수 있습니다. 너무 뻣뻣하거나 너무 굽혀지면 부상 위험이 증가합니다.

**판정 기준**

| 판정 | 의미 | 색상 |
|---|---|---|
| Stiff Knee | 무릎이 충분히 굽혀지지 않음 (충격 흡수 부족) | 🔴 |
| Good Flexion | 정상 범위 | 🟢 |
| Over Bent | 무릎이 과도하게 굽혀짐 | 🔴 |
| Borderline | 경계 구간 | 🟡 |

#### 지표 2-A: 발 착지 유형 (Foot Strike)

**측정 방식**: 발뒤꿈치와 발끝 좌표의 수평 기울기

**왜 중요한가**: 발의 어느 부분으로 먼저 착지하는지에 따라 충격 분산 방식이 달라집니다.

**판정 기준**

| 판정 | 의미 | 색상 |
|---|---|---|
| Heel Strike | 뒤꿈치부터 착지 (무릎 부담 큼) | 🔴 |
| Mid-Foot Strike | 중족부 착지 (가장 안정적) | 🟢 |
| Forefoot Strike | 앞꿈치 착지 (종아리 부담) | 🟡 |

#### 지표 2-B: 오버스트라이딩 (Overstriding)

**측정 방식**: 착지 시점의 발목과 골반의 수평 거리

**왜 중요한가**: 무게중심보다 발이 너무 앞에 착지하면 브레이크 효과로 무릎 부상 위험이 큽니다.

**판정 기준**

| 판정 | 의미 | 색상 |
|---|---|---|
| Over Stride | 무게중심 대비 발이 너무 앞 (브레이크 효과) | 🔴 |
| Good Stride | 적절한 보폭 | 🟢 |

#### 지표 3: 수직 진폭 (Vertical Oscillation)

**측정 방식**: 한 스트라이드 동안의 골반 상하 진폭

**왜 중요한가**: 러닝 시 상하 움직임이 클수록 에너지 낭비가 많고 무릎 충격이 증가합니다.

**판정 기준**

| 판정 | 의미 | 색상 |
|---|---|---|
| High Oscillation | 상하 움직임 과다 (에너지 낭비) | 🔴 |
| Good Oscillation | 효율적인 진폭 | 🟢 |

### 6.2 보조 지표: 좌/우 비대칭

좌/우 발의 착지 횟수, 평균 무릎 각도, 수직 진폭을 비교하여 **편측 부상 위험**을 조기에 탐지합니다.

**경고 기준**: 좌우 차이 10% 초과 시 🟡 경고 메시지 출력

## 9. 차별화 전략

### 9.1 차별점 1 — 100% 한국어 현지화

경쟁사 4곳 모두 영문 UI입니다. 한국 트레이너와 일반 사용자에게 가장 큰 진입 장벽이 언어입니다.

**적용 범위**
- 모든 화면 UI
- 분석 리포트 (PDF, 공유 링크)
- AI 코칭 메시지 (자연스러운 한국어 어투)
- 에러 메시지

### 9.2 차별점 2 — 트레이너 모드 토글 (One App, Two Roles)

별도 앱이나 별도 계정 없이 **토글 하나로** B2C ↔ B2B 전환됩니다.

**비교**
- Movaia: 코칭 계정과 회원 계정이 완전히 분리 → 트레이너가 본인 분석 시 별도 로그인 필요
- TreadForm: 트레이너가 본인의 러닝도 같은 앱에서 측정 → 자연스러운 통합 UX

### 9.3 차별점 3 — 직관성 우선 (Selection over Information)

**경쟁사 방식**: "더 많은 정보가 좋다" → 15개 이상 메트릭 노출
**TreadForm 방식**: "꼭 필요한 정보만 명확하게" → 3대 지표 + 색상 신호

**구체적 차이**

| 측면 | RunFormAI | TreadForm |
|---|---|---|
| 정보량 | 15+ 메트릭 | 3대 핵심 |
| 평가 방식 | 수치 점수 (73점) | 🔴🟡🟢 신호 |
| 문제 구간 표시 | 어노테이션 영상 | 자동 마커 + 슬로우 모션 |
| 사용자 부담 | 어디가 문제인지 직접 탐색 | 자동으로 강조됨 |

### 9.4 차별점 4 — 트레드밀 환경 특화

경쟁사들은 야외+실내 모두 커버하려다 변수 통제력이 떨어집니다.

**트레드밀 환경 특화의 이점**
- 외부 변수(조도, 거리, 배경) 통제로 분석 정확도 향상
- AR 가이드라인이 트레드밀 벨트 선과 정확히 매칭
- 국내 피트니스 센터 대부분이 트레드밀 구비 → 시장 적합성 높음
- 일정한 속도로 러닝하므로 분석 일관성 보장

## 부록 C. FAQ

**Q1. 왜 트레드밀 환경만 지원하나요?**

A. 외부 변수(조도, 거리, 배경)를 통제하여 분석 정확도를 높이기 위함입니다. 또한 국내 피트니스 센터의 대부분이 트레드밀을 보유하고 있어 시장 적합성이 가장 높습니다. 향후 야외 모드도 검토할 예정입니다.

**Q2. 분석에 얼마나 걸리나요?**

A. 약 30초 (10초 영상 기준). 노트북 CPU 환경 기준이며, 클라우드 이전 시 더 빨라질 수 있습니다.

**Q3. 한국어 코칭 메시지는 어떻게 생성되나요?**

A. 분석된 지표의 상태별 카운트를 기반으로 가장 심각한 문제를 우선순위화한 후, 한국어 템플릿에 데이터를 삽입하여 자연스러운 문장을 생성합니다.

**Q4. 회원 데이터는 안전한가요?**

A. 현재 프로젝트 단계에서는 로컬 PC에 저장됩니다. 창업 연계 시 암호화된 클라우드 저장소로 이전하며, 회원별 권한 분리 체계를 구축할 예정입니다.

**Q5. 다른 운동 분석도 가능한가요?**

A. 현재는 트레드밀 러닝에 특화되어 있습니다. 같은 AI 모델(MediaPipe Pose)을 활용하면 스쿼트, 데드리프트 등 다른 운동 분석으로 확장 가능하며 장기 로드맵에 포함되어 있습니다.

**Q6. 어떤 스마트폰에서 사용 가능한가요?**

A. iOS 13+ / Android 9+ 이상의 카메라가 있는 스마트폰이면 사용 가능합니다.

**Q7. 인터넷 연결이 필요한가요?**

A. 분석 서버와 통신해야 하므로 WiFi 또는 LTE/5G 연결이 필요합니다. 피트니스 센터 내 WiFi 활용을 권장합니다.
